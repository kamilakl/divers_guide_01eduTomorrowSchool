# Quest 6.4 Sort Params  

## Название файла Go  
`sortparams/main.go`

---
## Условия задания

| **Instruction** | **Перевод** |
|---------------|--------------|
| *Write a program that prints the arguments received on the command line in ASCII order.* | *Напишите программу, которая выводит аргументы, полученные в командной строке, в порядке ASCII.* |

---

## Пример вывода

```
$ go run . 1 a 2 A 3 b 4 C
1
2
3
4
A
C
a
b
$
```

---

## Решение

```go
package main

import (
	"os"
	"github.com/01-edu/z01"
)

func main() {
	if len(os.Args) > 1 {
		args := os.Args[1:]
		n := len(args)
		for i := 0; i < n-1; i++ {
			for j := 0; j < n-1-i; j++ {
				if args[j] > args[j+1] {
					args[j], args[j+1] = args[j+1], args[j]
				}
			}
		}
		for _, arg := range args {
			for _, r := range arg {
				z01.PrintRune(r)
			}
			z01.PrintRune('\n')
		}
	}
}
```
---

## Ключевые понятия

| **Термин** | **Объяснение** |
|-------------|----------------|
| **ASCII порядок** | Символы сортируются по их числовому коду в таблице ASCII. Цифры идут первыми (48–57), затем заглавные буквы (65–90), потом строчные (97–122). <br>Подробнее: [ASCII таблица](https://www.asciitable.com) |
| **Bubble Sort (сортировка пузырьком)** | Простой алгоритм сортировки с временной сложностью `O(n²)`. Для небольших массивов удобен и нагляден. |
| **os.Args** | Срез строк, представляющий аргументы командной строки. Первый элемент — имя программы. |
| **Сравнение строк в Go** | Выполняется по кодам Unicode символов. `"A" < "a"` потому что 65 < 97. |
| **z01.PrintRune** | Функция из библиотеки `z01`, которая выводит одну руну (символ) на экран. Используется вместо `fmt.Print`. |

## Логика и пошаговый разбор кода

### 1. Проверка наличия аргументов

```go
package main

import (
	"os"
	"github.com/01-edu/z01"
)

func main() {
	if len(os.Args) > 1 {
```

Проверяем, передал ли пользователь аргументы при запуске программы.  
Если аргументов нет (`len(os.Args) == 1`), программа ничего не делает.  
Если аргументы есть — продолжаем выполнение.

---

### 2. Извлечение аргументов без имени программы

```go
args := os.Args[1:]
```
`os.Args` это срез строк (`[]string`), где первый элемент (`os.Args[0]`) это имя исполняемого файла.  
Используя `os.Args[1:]`, мы исключаем его, чтобы работать только с переданными аргументами.

Пример:
```
go run . a B c
→ os.Args = ["main", "a", "B", "c"]
→ args = ["a", "B", "c"]
```
---
### 3. Алгоритм сортировки пузырьком (Bubble Sort)

```go
n := len(args)
for i := 0; i < n-1; i++ {
	for j := 0; j < n-1-i; j++ {
		if args[j] > args[j+1] {
			args[j], args[j+1] = args[j+1], args[j]
		}
	}
}
```
Принцип работы:  
Сравниваем соседние элементы `args[j]` и `args[j+1]`.  
Если левый элемент больше правого, меняем их местами.  

Вложенные циклы гарантируют, что после каждого прохода самый большой элемент всплывает в конец списка как пузырёк.

Пошагово:
1. Внешний цикл `i` управляет количеством проходов.  
2. Внутренний цикл `j` сравнивает элементы попарно.  
3. `if args[j] > args[j+1]` — сравнение по ASCII-порядку.  

Bubble Sort алгоритм имеет time complexity O(n²), что может быть неэффективно при большом количестве аргументов.

---

### 4. Печать отсортированных аргументов

```go
for _, arg := range args {
	for _, r := range arg {
		z01.PrintRune(r)
	}
	z01.PrintRune('\n')
}
```
После сортировки, каждый аргумент выводится на новой строке:
- Внешний цикл перебирает все аргументы.
- Внутренний цикл выводит каждый символ аргумента по одному, используя z01.PrintRune(r) для вывода символов.
- После каждого аргумента печатается символ новой строки (\n), чтобы аргументы выводились на отдельных строках.
